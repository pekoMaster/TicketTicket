# 管理員後台系統

## 功能描述
建立完整的管理員後台系統，提供刊登管理、會員管理和黑名單功能，讓管理員能有效管理平台內容和用戶。

## 問題陳述
目前管理員後台只有活動（Event）管理功能，缺乏：
1. **刊登管理**：無法直接管理用戶發布的票券刊登，需要介入時沒有工具
2. **會員管理**：無法處理違規用戶的暱稱或頭像
3. **黑名單系統**：無法封鎖惡意用戶，使其無法使用平台

## 提議解決方案

### 1. 刊登管理頁面 (`/admin/listings`)
- 列表形式顯示所有刊登（非卡片形式）
- 支援搜尋、篩選（狀態、日期）、排序
- 強制編輯刊登內容
- 強制移除刊登
- 移除/編輯時可選擇是否通知相關用戶（主辦方、所有申請者）
- 通知透過系統訊息發送

### 2. 會員管理頁面 (`/admin/users`)
- 列表形式顯示所有會員
- 支援搜尋（暱稱、Email）
- 強制更改暱稱
- 強制更改/移除頭像
- 加入/移出黑名單
- 查看用戶刊登歷史和評價

### 3. 黑名單系統
- 新增 `blacklist` 資料表
- 封鎖條件：Google 帳號 Email
- 被封鎖用戶無法登入和註冊
- 封鎖時可填寫原因
- 支援解除封鎖

### 4. 管理員通知系統
- 管理員操作時可選擇是否通知用戶
- 通知內容：說明操作原因
- 通知方式：系統訊息（conversations 中的特殊訊息）

## 成功標準
- [ ] 管理員可在列表中查看和搜尋所有刊登
- [ ] 管理員可強制編輯/移除刊登，並選擇是否通知用戶
- [ ] 管理員可在列表中查看和搜尋所有會員
- [ ] 管理員可強制更改會員暱稱和頭像
- [ ] 管理員可將用戶加入/移出黑名單
- [ ] 被封鎖用戶無法登入或註冊
- [ ] 所有管理操作有日誌記錄

## 範圍外
- 管理員權限分級（只有單一管理員角色）
- Email 通知（只使用站內訊息）
- 自動封鎖規則（全手動操作）
- IP 封鎖（只封鎖 Google 帳號）
